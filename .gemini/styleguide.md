# Руководство по стилю кода — ToDeskAI Discovery

## Язык — КРИТИЧЕСКИ ВАЖНО
- **ВСЕ свои ответы, комментарии к PR, саммари и ревью пиши ТОЛЬКО НА РУССКОМ ЯЗЫКЕ**
- Все комментарии в коде, документация, логи и сообщения об ошибках — ТОЛЬКО на русском языке
- Идентификаторы кода (переменные, функции, типы) — на английском
- Ревью-комментарии пиши на русском
- **Никогда не пиши на английском** — весь текст от тебя должен быть на русском

## Архитектура
- Проект: Rust сервер обнаружения (Discovery) для ToDeskAI
- Cargo workspace

## Rust
- Именование полей в API: camelCase через `#[serde(rename_all = "camelCase")]`
- Обработка ошибок через `Result<T, E>` с понятными сообщениями на русском

## Безопасность
- Проверять на уязвимости OWASP Top 10
- Не допускать утечки секретов, API-ключей, токенов
- Валидация входных данных на границах системы

## На что обращать внимание при ревью
- Корректность логики и обработка edge cases
- Потенциальные утечки памяти и ресурсов
- Правильная обработка ошибок
- Безопасность
- Соответствие стилю кода проекта
